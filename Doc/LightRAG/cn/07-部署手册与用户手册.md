# 第一部分：部署手册

## 一、环境要求

### 1.1 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux / macOS / Windows（跨平台） |
| Python | ≥ 3.10 |
| 包管理 | uv（推荐）或 pip |
| 内存 | 视语料规模，建议 ≥ 4GB |
| 磁盘 | 存放工作目录、向量库、图数据 |

### 1.2 依赖组件（按需）

| 组件 | 用途 | 何时需要 |
|------|------|----------|
| OpenAI/兼容 LLM 端点 | 生成与抽取 | 默认 |
| Ollama | 本地 LLM/嵌入 | 离线/本地 |
| PostgreSQL + pgvector + AGE | 四合一存储 | 生产统一后端 |
| MongoDB | 四合一存储 | 文档型统一后端 |
| Neo4j | 图存储 | 大规模图谱 |
| Milvus / Qdrant / FAISS | 向量存储 | 大规模向量 |
| Redis | KV/状态缓存 | 分布式 |
| OpenSearch | 四合一存储 | 检索型统一后端 |
| MinerU / Docling | 多模态解析 | 多模态文档 |

## 二、安装

### 2.1 uv（推荐）
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh          # Unix/macOS
# Windows: powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

uv pip install -e ".[api]"                                # API 服务 + 文档解析
```

### 2.2 安装可选能力
```bash
uv pip install -e ".[api]"            # Web API + 文档处理
uv pip install -e ".[offline-storage]" # 全部存储后端
uv pip install -e ".[offline-llm]"     # 全部 LLM 提供商
uv pip install -e ".[offline]"         # 完整离线包（=api+storage+llm）
uv pip install -e ".[evaluation]"      # RAGAS 评估
uv pip install -e ".[observability]"   # Langfuse 追踪
```

## 三、配置项说明

配置优先级：**环境变量 > 构造参数 > config.ini（已弃用）**。推荐使用 `.env`（参考根目录 `env.example`，56KB 含全部变量）。

### 3.1 核心配置

| 变量 | 说明 | 示例 |
|------|------|------|
| `WORKSPACE` | 工作区名 | `default` |
| `WORKING_DIR` | 工作目录 | `./rag_storage` |
| `LLM_BINDING` | LLM 提供商 | `openai` / `ollama` / `azure_openai` / `bedrock` / `gemini` / `lollms` |
| `LLM_BINDING_HOST` | LLM 端点 | `http://localhost:11434/v1` |
| `LLM_MODEL` | 模型名 | `gpt-4o-mini` |
| `LLM_API_KEY` | API Key | `sk-...` |
| `EMBEDDING_BINDING` | 嵌入提供商 | `openai` / `ollama` / `jina` / `voyageai` / `bedrock` / `gemini` |
| `EMBEDDING_MODEL` | 嵌入模型 | `text-embedding-3-large` |
| `EMBEDDING_DIM` | 嵌入维度 | `3072` |
| `MAX_ASYNC_LLM` | LLM 并发上限 | `4` |
| `DEFAULT_LLM_TIMEOUT` | LLM 超时（秒） | `120` |

### 3.2 角色化 LLM（EXTRACT/QUERY/KEYWORDS/VLM）
可为四种角色独立配置 `host/model/api_key/max_async`：
```env
LLM_BINDING_HOST=...        # 全局默认
# 角色覆盖（前缀）：
EXTRACT_LLM_MODEL=...
QUERY_LLM_BINDING_HOST=...
KEYWORDS_LLM_MODEL=...
VLM_LLM_MODEL=...
```

### 3.3 存储后端
```env
KV_STORAGE=JsonKVStorage                # 或 PGKVStorage / MongoKVStorage / RedisKVStorage / OpenSearchKVStorage
VECTOR_STORAGE=NanoVectorDBStorage      # 或 MilvusVectorDBStorage / PGVectorStorage / FaissVectorDBStorage / QdrantVectorDBStorage
GRAPH_STORAGE=NetworkXStorage           # 或 Neo4JStorage / PGGraphStorage / MongoGraphStorage / MemgraphStorage
DOC_STATUS_STORAGE=JsonDocStatusStorage # 对应实现
```
对应连接变量：`POSTGRES_*`、`MONGO_*`、`NEO4J_*`、`MILVUS_*`、`QDRANT_*`、`REDIS_URI`、`OPENSEARCH_HOSTS` 等。

### 3.4 分块与查询
```env
CHUNK_SIZE=1200
CHUNK_F_OVERLAP_SIZE=100
CHUNKING_STRATEGY=chunking/by_token_size   # 或 recursive / semantic_vector / paragraph
TOP_K=60
COSINE_THRESHOLD=0.2
RERANK_MODEL=...                           # 启用重排
```

### 3.5 安全
```env
LIGHTRAG_API_KEY=...              # 多用户鉴权开关
TOKEN_SECRET_KEY=...              # JWT 密钥
LIGHTRAG_API_ACCESS_CONTROL=...   # 访问控制
```

## 四、部署步骤

### 4.1 本地直接运行
```bash
# 1. 准备环境变量
cp env.example .env        # 按需修改

# 2. 启动服务
lightrag-server --host 0.0.0.0 --port 9621

# 或 Gunicorn（生产）
lightrag-gunicorn
```

### 4.2 Docker
```bash
docker compose up -d                       # 基础（docker-compose.yml）
docker compose -f docker-compose-full.yml up -d   # 含 Postgres + Neo4j 等全栈
```
- 镜像：`Dockerfile`（完整）、`Dockerfile.lite`（精简）、`Dockerfile.postgres`（PG 一体）
- Podman：`docker-compose.podman.yml`
- 入口脚本：`docker-entrypoint.sh`

### 4.3 Kubernetes
- 清单目录：`k8s-deploy/`（`lightrag/`、`databases/`）
- 按需 apply：`kubectl apply -f k8s-deploy/`

### 4.4 systemd
- 参考 `lightrag.service.example`

### 4.5 Setup Wizard（本地一键）
- 向导支持本地 Docker 部署嵌入、重排、存储后端

# 第二部分：用户手册

## 一、功能操作指南

### 1.1 通过 Web UI
1. 浏览器访问 `http://<host>:9621/webui/`
2. 登录（如启用鉴权）
3. **文档管理**：上传文件 / 输入文本 / 扫描目录
4. **知识库查询**：选择查询模式（local/global/hybrid/mix/naive），输入问题
5. **知识图谱**：可视化浏览实体与关系，支持编辑/合并/删除
6. **追踪**：查看文档处理流水线状态

### 1.2 通过 REST API
```bash
# 摄入文本
curl -X POST http://localhost:9621/documents/text \
  -H "Authorization: Bearer <JWT>" -H "Content-Type: application/json" \
  -d '{"text":"...","track_id":"t-001"}'

# 查询（hybrid 模式）
curl -X POST http://localhost:9621/query \
  -H "Authorization: Bearer <JWT>" -H "Content-Type: application/json" \
  -d '{"query":"...","mode":"hybrid"}'
```

### 1.3 通过 Python SDK
```python
import asyncio
from lightrag import LightRAG, QueryParam
from lightrag.llm import openai_complete_if_cache, openai_embed

async def main():
    rag = LightRAG(working_dir="./rag_storage")
    await rag.initialize_storages()
    await rag.ainsert("你的文档内容...")
    ans = await rag.aquery("你的问题", param=QueryParam(mode="hybrid", stream=True))
    async for chunk in ans:
        print(chunk, end="")

asyncio.run(main())
```

## 二、配置说明（用户可调）

| 配置 | 作用 | 建议 |
|------|------|------|
| 查询模式 | 控制检索范围 | 细节问题用 local，综述用 global，默认 mix |
| `top_k` | 召回数量 | 越大越全但更慢 |
| Rerank | 重排提升精度 | mix 模式建议开启 |
| `history_turns` | 多轮对话上下文 | 对话场景设 1-3 |
| 分块策略 | chunk 粒度 | 通用 token_size；结构化文档用 paragraph |
| `chunk_size` | 块大小 | 1200（默认），影响召回精度 |
| 角色化 LLM | 不同环节用不同模型 | 抽取用强模型，关键词用快模型 |

## 三、常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| LLM 超时 | 端点慢/网络 | 调大 `DEFAULT_LLM_TIMEOUT`；见 docs 超时排障 |
| 摄入卡住 | 并发过高/限流 | 降低 `MAX_ASYNC_LLM`、`max_parallel_insert` |
| LLM 输出非法 JSON | 模型能力弱 | 开启 `entity_extraction_use_json`；用更强模型；`json_repair` 自动修复 |
| 向量维度不匹配 | 切换嵌入模型 | `lightrag-rebuild-vdb` 重建向量库 |
| 图谱为空 | 摄入未完成 | 查 `/documents/pipeline_status`、`track_status` |
| 查询不准 | 模式/重排未调 | 换 mode；开启 rerank；调 `top_k`/`cosine_threshold` |
| 存储连接失败 | 环境变量缺失 | 检查对应后端的连接变量（见 3.3） |
| 多 Worker 数据不一致 | 未等索引完成 | 依赖 `shared_storage` + 完成回调 |
| 鉴权 401 | JWT 过期/缺失 | 重新 `POST /login` |
| Bun 'bin metadata file' 错误（WSL） | WebUI 构建 | 见 docs/webui 文档说明 |
| 内存占用高 | 语料大 | 改用 PG/Milvus 等外置后端 |
| 磁盘增长 | 缓存累积 | 定期 `lightrag-clean-llmqc` |

## 四、安全须知
- 对外暴露服务时**必须启用鉴权**（`LIGHTRAG_API_KEY` + JWT）
- 密码使用 bcrypt：`lightrag-hash-password` 生成哈希
- 密钥、Token 通过环境变量注入，勿入仓
- 内网/合规场景使用全本地化部署（Ollama + 本地存储），数据不出域

## 五、离线/内网部署
- 参考 `docs/OfflineDeployment.md`
- 预装依赖：`[offline]`、`[offline-storage]`、`[offline-llm]`
- 预下载缓存：`lightrag-download-cache`
