## 一、项目概述

### 1.1 项目背景
LightRAG 是由香港大学数据智能实验室（HKUDS）开源的轻量级检索增强生成（RAG）框架。它将大语言模型（LLM）与**知识图谱（Knowledge Graph）**深度融合，通过**双层检索范式（Dual-Level Retrieval）**——低层实体/关系检索与高层主题/关键词检索——克服了传统向量检索 RAG 在复杂推理、多跳问答和全局视角上的不足。

### 1.2 核心业务目标
- 将异构文档（TXT、PDF、DOCX、PPTX、XLSX、Markdown、图片等）自动解析、分块并构建为知识图谱
- 支持多策略语义检索与生成式问答，覆盖局部细节到全局主题
- 提供可插拔的存储后端（KV / 向量 / 图 / 文档状态），适配从本地单机到分布式集群的部署场景
- 提供开箱即用的 RESTful API 服务与可视化 Web 管理界面

### 1.3 适用范围
面向需要在私有/本地知识库之上构建智能问答、知识管理、文档分析系统的开发者和企业用户。

## 二、功能需求清单

### 2.1 文档摄入与解析（Ingestion & Parsing）
- **文档上传**：支持通过 API 上传文件、纯文本、批量文本，以及扫描指定目录
- **多格式解析**：内置原生解析器支持 TXT、MD、HTML（含表格提取）、CSV/JSON；集成层支持 PDF、DOCX、PPTX、XLSX、SVG 等
- **多模态解析**（RAG-Anything / MinerU / Docling 集成）：处理图片、表格、公式等多模态内容
- **文档追踪**：为每个摄入任务分配 `track_id`，可查询异步处理流水线状态

### 2.2 文本分块（Chunking）
提供四种可选分块策略：
- **Fix（Token Size）**：按固定 token 数量切分（默认 `token_size=1200`）
- **Recursive（递归字符）**：基于分隔符层级递归切分（langchain-text-splitters）
- **Vector（语义向量）**：基于嵌入相似度的语义切分（SemanticChunker）
- **Paragraph（段落）**：按文档段落结构切分

### 2.3 知识图谱构建（Knowledge Graph Construction）
- **实体与关系抽取**：调用 LLM 从文本块中抽取实体、关系及关键 claim
- **增量合并**：对新文档抽取结果与已有图谱进行去重、合并与置信度更新
- **自定义 KG 注入**：支持直接注入预先构建好的实体/关系数据（`insert_custom_kg`）
- **大模型友好**：针对开源 LLM（如 Qwen3）优化抽取提示词，提升图构建准确率

### 2.4 检索与问答（Retrieval & Query）
支持六种查询模式（`QueryParam.mode`）：
| 模式 | 说明 |
|------|------|
| `local` | 聚焦具体实体、局部上下文相关的事实 |
| `global` | 利用全局知识、主题层面汇总 |
| `hybrid` | 同时使用低层（实体）与高层（关键词）检索 |
| `naive` | 传统向量召回（不使用知识图谱） |
| `mix` | 整合知识图谱检索与向量检索（**默认模式**） |
| `bypass` | 跳过检索，直接将问题转交 LLM |

- **流式输出**：支持 SSE 流式返回生成结果（`/query/stream`）
- **Reranker 重排**：内置重排序器，显著提升混合查询性能
- **引用溯源（Citation）**：返回结果可附带来源文档与块引用
- **关键词提取**：从查询中抽取用于图检索的高层/低层关键词
- **上下文返回**：API 可同时返回检索到的上下文，便于 RAGAS 精度评估

### 2.5 知识图谱管理（Graph Management）
- 实体/关系的查询、创建、编辑、删除、合并
- 实体存在性检查、标签（label）列表/热度/搜索
- 按文档删除（`delete_by_doc_id`）并自动重连受影响的关系
- 图谱可视化数据导出（节点、边）

### 2.6 存储与后端（Storage & Backends）
- **四大存储抽象**：KV 存储、向量存储、图存储、文档状态存储
- **后端矩阵**：JSON / Redis / PostgreSQL / MongoDB / OpenSearch（KV）；NanoVectorDB / Milvus / PostgreSQL+pgvector / FAISS / Qdrant / MongoDB / OpenSearch（向量）；NetworkX / Neo4j / PostgreSQL(AGE) / MongoDB / Memgraph / OpenSearch（图）
- **一键存储**：PostgreSQL、MongoDB、OpenSearch 可作为四合一统一后端
- **存储迁移**：提供 `storage_migrations` 与 `rebuild_vdb` 工具

### 2.7 LLM 与嵌入模型（LLM & Embedding）
- **角色化 LLM 配置**：为 EXTRACT / QUERY / KEYWORDS / VLM 四种角色独立配置模型、端点、密钥
- **多提供商**：OpenAI / Azure OpenAI / Anthropic / Gemini / Ollama / HuggingFace / Bedrock / LM Deploy / NVIDIA / VoyageAI / Jina / 智谱(Zhipu) / LlamaIndex 等
- **本地化部署**：通过 Setup Wizard 支持本地 Docker 部署嵌入、重排、存储后端

### 2.8 API 服务与 Web UI
- **RESTful 服务**：FastAPI 实现的查询、文档、图谱、Ollama 兼容接口
- **认证鉴权**：JWT + 用户名密码（bcrypt 加密），可配置单/多用户
- **Web 管理界面**：基于 React/Vite 的前端，支持插入、查询、图谱可视化、文档管理

### 2.9 可观测性与评估
- **Langfuse 集成**：LLM 调用链路追踪
- **RAGAS 评估**：基于检索上下文计算精度等指标

## 三、业务规则需求

### 3.1 摄入幂等与去重
- 同一文档重复摄入应通过文档状态存储进行幂等控制，避免重复构建
- 文档状态机：`PENDING → PROCESSING → PROCESSED`（或 `FAILED`），失败可重处理（`/reprocess_failed`）

### 3.2 并发与流水线控制
- 支持并发摄入与并发实体抽取（`entity_extract_max_gleaning`、并发数可配）
- 流水线可取消（`/cancel_pipeline`）
- 多 Worker 部署下，共享存储需保证索引一致（`shared_storage` 机制）

### 3.3 查询行为规则
- 混合查询默认启用 Reranker
- 历史对话上下文（`history_turns`）可作为查询输入增强多轮对话
- 生成结果引用需可追溯到原始文档块

### 3.4 安全规则
- API 服务在多用户模式下默认强制鉴权
- 凭证、密钥通过环境变量注入，禁止硬编码
- 服务对外暴露时必须启用认证（部署手册含安全告警）

### 3.5 数据生命周期
- 删除文档时应同步清理对应的实体、关系与向量
- 缓存（LLM 查询缓存、向量缓存）可独立清理（`/clear_cache`、`clean_llm_query_cache` 工具）

## 四、非功能需求

### 4.1 性能要求
- 支持大规模数据集高效处理（消除处理瓶颈的扩展性优化）
- 流式响应首 token 延迟应满足交互式问答体验
- 向量召回、图检索支持批量接口（`get_nodes_batch`、`get_vectors_by_ids`）降低 IO

### 4.2 可扩展性
- 存储后端、LLM 提供商、解析器、分块器均通过工厂模式 + 注册表实现热插拔
- 支持水平扩展：PostgreSQL / Milvus / Neo4j / Redis 等生产级后端可独立扩容

### 4.3 可靠性与容错
- LLM 调用具备重试机制（tenacity）与超时配置
- 文件原子写入（`file_atomic`）防止并发写入损坏
- 处理失败的任务可追踪与重试

### 4.4 兼容性
- Python ≥ 3.10
- 跨平台：Linux / macOS / Windows，支持 Docker / Podman / Kubernetes 部署
- 离线/内网部署：提供 `offline`、`offline-storage`、`offline-llm` 依赖分层

### 4.5 安全要求
- 数据安全：支持全本地化部署（数据不出域）
- 访问控制：JWT 鉴权、bcrypt 密码哈希
- 依赖安全：对 transitive 依赖（python-jose、starlette、python-multipart）设置安全版本下限

### 4.6 可维护性
- 配置通过 `config.ini`、`.env`、构造参数三路注入
- 提供版本管理、变更日志、存储迁移工具
- 代码遵循 ruff 规范，pytest 异步测试体系

### 4.7 国际化
- README 提供中文、英文、日文版本
- Web UI 多语言支持
