## 五、测试数据准备

- **语料集**：项目内置 `reproduce/`、`prompts/samples/` 提供示例文本；可使用 Dickens 等公开语料
- **Mock LLM**：测试中对 LLM 调用打桩（固定 JSON 响应），避免真实 API 消耗
- **Mock Embedding**：固定维度随机/确定性向量
- **临时工作区**：每个测试用例使用独立 `working_dir`，结束后清理
- **RAGAS 评估集**：`tests/evaluation/` + `lightrag/evaluation/eval_rag_quality.py`，准备 query-relevant-context 三元组
- **存储后端测试环境**：PostgreSQL/MongoDB/Neo4j/Milvus 等可通过 docker-compose 起本地实例

## 六、评估测试（RAGAS）

- **指标**：上下文精度（context precision）、答案相关性、忠实度（faithfulness）

- **依赖**：`pip install -e ".[evaluation]"`（ragas、datasets）
