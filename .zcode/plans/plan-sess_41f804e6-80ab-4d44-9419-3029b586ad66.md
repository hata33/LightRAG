# 检索进度推送功能实现方案

## 目标
在查询流式响应中,推送检索管线的进度事件(提取关键词→检索实体→检索关系→检索文本块→重排→生成回复),让前端实时显示执行到了哪一步。

## 架构设计

### 核心思路:callback + asyncio.Queue 合并生成器

```
query_text_stream (路由层)
  │
  ├─ progress_queue = asyncio.Queue()
  ├─ task = create_task(aquery_llm(..., progress_callback=cb))
  │      cb = lambda event: progress_queue.put_nowait({"progress": event})
  │
  └─ merged_generator:
       1. 先 drain progress_queue (while task not done) → yield {"progress":"..."} NDJSON
       2. task 完成后 → yield references + response chunks + response_time
```

检索跑在 task 里,生成器同时 drain 队列输出进度。检索完成后 task 返回 result,生成器切换到输出 LLM 响应流。

### 3. 后端路由 — `lightrag/api/routers/query_routes.py`

**(a) 重构 `query_text_stream` 的流式路径**(line 753-782)

当前:
```python
result = await rag.aquery_llm(...)  # 阻塞
stream_gen = _build_stream_generator(result=result, ...)
return StreamingResponse(stream_gen(), ...)
```

改为:
```python
progress_queue = asyncio.Queue()

async def progress_callback(event: str):
    await progress_queue.put({"progress": event})

start_time = time.perf_counter()
task = asyncio.create_task(rag.aquery_llm(..., progress_callback=progress_callback))

async def merged_generator():
    # Phase 1: drain progress events while retrieval runs
    while not task.done():
        try:
            event = await asyncio.wait_for(progress_queue.get(), timeout=0.1)
            yield f"{json.dumps(event)}\n"
        except asyncio.TimeoutError:
            continue
    # Task is done — check for exceptions
    result = await task  # raises if task failed
    
    # Phase 2: drain any remaining progress events
    while not progress_queue.empty():
        event = progress_queue.get_nowait()
        yield f"{json.dumps(event)}\n"
    
    # Phase 3: yield references + LLM response chunks + response_time
    # (reuse existing _build_stream_generator logic inline or call it)
    ...

return StreamingResponse(merged_generator(), ...)
```

**(b) `_build_stream_generator`** 不需要改动 — merged_generator 在进度事件输出完后,内联调用其生成逻辑(或提取为可复用的 async generator)。

**(c) `/query` 非流式端点** — 不加进度推送(非流式无法增量返回)。但 `aquery_llm` 仍接受 callback,只是非流式端点不传。

## 进度事件顺序(mix 模式)

```
{"progress":"extracting_keywords"}      ← Keyword LLM 调用前
{"progress":"retrieving_entities"}      ← 向量检索实体前
{"progress":"retrieving_relations"}     ← 向量检索关系前
{"progress":"retrieving_chunks"}        ← 向量检索文本块前
{"progress":"reranking"}                ← rerank API 调用前
{"progress":"generating_response"}      ← LLM 生成前
{"references":[...]}                    ← 引用列表
{"response":"chunk1"}                   ← LLM 响应流
{"response":"chunk2"}
{"response_time":9.6}                   ← 最终耗时
```

## 不改动的部分
- `/query` 非流式端点不加进度(无法增量返回)
- `/query/data` 端点(仅检索无 LLM)
- Ollama 兼容 API(有自己的格式)
- `aquery`/`aquery_data` 方法签名(只在 `aquery_llm` 加可选参数)

## 风险评估
- **侵入性中等**:改 operate.py 的核心检索函数签名,但 `progress_callback` 是可选参数(default None),不影响现有调用者
- **asyncio.Task 异常处理**:merged_generator 必须 `await task` 捕获异常并转为 error NDJSON 行
- **超时 drain**:用 `asyncio.wait_for(queue.get(), timeout=0.1)` 避免在 task 完成后卡在空队列上
